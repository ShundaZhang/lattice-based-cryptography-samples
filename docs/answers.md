# 习题答案

## A. 线性代数和模运算基础

### 1. 向量加法

```text
(2, 3) + (5, -1) = (7, 2)
```

### 2. 向量数乘

```text
4 * (2, -3, 1) = (8, -12, 4)
```

### 3. 点积

```text
7*4 + 3*8 + 11*1 = 28 + 24 + 11 = 63
```

### 4. 范数

```text
||(3,4)|| = sqrt(3^2 + 4^2) = 5
||(3,4)||^2 = 25
```

### 5. 模运算

```text
23 mod 17 = 6
-3 mod 17 = 14
8*9 = 72 = 4 mod 17
```

### 6. 中心化模表示

在 `Z_97` 中：

```text
96 -> -1
95 -> -2
48 -> 48
49 -> -48
```

### 7. 行列式

```text
det [[4,1],[1,3]] = 4*3 - 1*1 = 11
```

几何意义是基本平行四边形面积为 `11`，也可以理解为这个二维格的单位格点密度指标。

### 8. 矩阵乘向量

```text
第一行: 1*5 + 2*6 = 17 = 0 mod 17
第二行: 3*5 + 4*6 = 39 = 5 mod 17
```

所以结果是：

```text
(0, 5)
```

### 9. 线性组合

```text
3*(2,1) - 1*(1,3) = (6,3) - (1,3) = (5,0)
```

### 10. 同一个格的不同基

`(1,0)` 已经能生成所有水平整数移动。
又因为：

```text
(0,1) = (100,1) - 100*(1,0)
```

所以这组基也能生成标准基 `(1,0),(0,1)`，因此能生成整个 `Z^2`。

### 11. 暴力搜索规模

候选数量是：

```text
q^n
```

当 `n=4, q=97` 时：

```text
97^4 = 88,529,281
```

### 12. 解密阈值

```text
q/4 = 97/4 = 24.25
```

粗略说，噪声绝对值最好明显小于 `24`。

## B. 格和困难问题

### 13. 格点计算

```text
2*(2,1) - 3*(1,2) = (4,2) - (3,6) = (1,-4)
```

### 14. 枚举短向量

非零短向量之一是：

```text
(1,3)
```

它的平方范数是：

```text
1^2 + 3^2 = 10
```

`(-1,-3)` 同样短。

### 15. 基本域面积

determinant 为 `5` 的格更密。
二维里 determinant 是基本平行四边形面积，面积越小，单位面积中格点越多。

### 16. 最近整数点

对每个坐标四舍五入：

```text
(2.7, -1.2) -> (3, -1)
```

所以最近标准格点是 `(3,-1)`。

### 17. BDD 直觉

`(3.1,4.2)` 离 `(3,4)` 很近，所以可以看成格点 `(3,4)` 加了小噪声 `(0.1,0.2)`。

### 18. SVP 定义

如果不要求非零，最短向量永远是零向量，问题没有意义。SVP 要找的是格中非零的最短向量。

### 19. CVP 与 SVP

SVP 是找离原点最近的非零格点；CVP 是给定任意目标点，找离它最近的格点。

### 20. 好基和坏基

好基短且接近正交，坐标方向清楚。用它近似最近点时，投影和四舍五入更可靠。坏基方向纠缠，坐标可能很大，近似最近点容易选错。

### 21. Unimodular 操作

用一个基向量减去另一个基向量的整数倍，只是在整数坐标系统中换一种表达方式。新的基向量仍然是旧基的整数线性组合，旧基也能用新基整数线性组合回来，所以格点集合不变。

### 22. Size reduction

通常希望：

```text
|mu[i][j]| <= 1/2
```

直觉是减少后一个向量在前一个 Gram-Schmidt 方向上的过大投影，让基更短、更接近正交。

### 23. Lovasz 条件

Lovasz 条件不满足说明当前局部顺序不够好，后面的向量可能比前面的方向更适合作为较早基向量。交换相邻向量后再规约，可能得到更短、更稳定的基。

### 24. BKZ

block size 越大，BKZ 越强，输出基质量通常越好，攻击能力越强；但计算成本也急剧上升。

## C. LWE、SIS 和 toy 加密

### 25. 计算一个 LWE 样本

```text
<a,s> = 3*4 + 5*2 = 22
b = 22 - 1 = 21 = 4 mod 17
```

所以 `b=4`。

### 26. 无噪声 LWE

如果 `e_i=0`，样本就是普通模线性方程：

```text
b_i = <a_i, s> mod q
```

收集足够多线性无关样本后，可以用线性代数解出 `s`，所以不安全。

### 27. 解码 phase 1

`3` 很接近 `0`，所以解成 bit `0`。

### 28. 解码 phase 2

`46` 接近 `q/2 = 48`，所以解成 bit `1`。

### 29. 解密失败

阈值约为 `24`。phase `18` 会被判断为更接近 `0`，所以 bit `1` 会被错误解成 bit `0`。

### 30. Toy LWE 正确性推导

因为 `b = A s + e`，所以：

```text
v - <s,u>
= b^T r + bit*floor(q/2) - <s, A^T r>
= (A s + e)^T r + bit*floor(q/2) - s^T A^T r
= e^T r + bit*floor(q/2)
```

如果 `e^T r` 足够小，就能判断结果更接近 `0` 还是 `floor(q/2)`。

### 31. 噪声大小取舍

噪声太小，样本接近普通线性方程，可能更容易被攻击。噪声太大，解密 phase 会跨过阈值，导致正确接收者也解不回来。

### 32. Search-LWE 与 Decision-LWE

Search-LWE 要恢复秘密 `s`。
Decision-LWE 要判断给定样本是 LWE 样本，还是完全随机样本。

### 33. SIS 小例子

```text
A*x = 1*1 + 2*(-2) + 3*1 = 1 - 4 + 3 = 0
```

所以：

```text
A*x = 0 mod 7
```

### 34. SIS 为什么强调短向量

模线性方程通常有很多解。如果不要求短，找到某个解可能并不难。SIS 的困难性来自“既要满足模方程，又要范数很小”。

### 35. Module-LWE

普通 LWE 结构少但密钥和矩阵大。Ring-LWE 效率高但结构假设更强。Module-LWE 使用多个环元素组成的模块，在效率和保守性之间折中。

### 36. 公钥和私钥

在本仓库 toy LWE 中：

- 公钥包含 `params, A, b`。
- 私钥包含 `params, s, e`。

真实方案通常不会把 `e` 作为解密必需私钥保存；toy 代码保存它主要是为了教学和观察噪声。

## D. 应用、标准和工程安全

### 37. KEM 接口

```text
keygen -> pk, sk
encaps(pk) -> ciphertext, shared_secret
decaps(sk, ciphertext) -> shared_secret
```

### 38. KEM 与 PKE

PKE 直接加密消息：

```text
encrypt(pk, message) -> ciphertext
decrypt(sk, ciphertext) -> message
```

KEM 只封装共享密钥，不直接加密任意长消息。

### 39. 标准任务

ML-KEM 用于密钥封装和密钥建立。
ML-DSA 用于数字签名、认证和完整性保护。

### 40. SLH-DSA

SLH-DSA 不是格基方案，它是 stateless hash-based signature。它在后量子签名标准中作为基于不同数学假设的保守备选。

### 41. Falcon/FN-DSA

Falcon/FN-DSA 是基于 NTRU lattice 思想的签名方向，使用 NTRU trapdoor 和高斯采样，优点是签名紧凑，难点是实现复杂。

### 42. FHE 噪声

FHE 密文带噪声，加法和乘法都会让噪声增长。噪声太大时无法正确解密。Bootstrapping 同态地刷新密文，把高噪声密文变成低噪声密文，从而支持更深计算。

### 43. 应用匹配

```text
TLS 建立会话密钥 -> KEM
固件升级完整性校验 -> 数字签名
云端在密文上做统计 -> 全同态加密
```

### 44. 多项式环计算

```text
x^3 * x^2 = x^5
```

在 `Z_q[x]/(x^4 + 1)` 中，`x^4 = -1`，所以：

```text
x^5 = x*x^4 = -x
```

结果是 `-x`，或在 `mod q` 下写成 `(q-1)*x`。

脚本是：

```text
examples/04_tiny_lwe_attack.py
```

它暴力枚举所有 secret。候选数是 `q^n`，真实参数下这个数会天文级增长，所以只能用于极小教学参数。
