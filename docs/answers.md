# 习题答案

## 1. 格点计算

```text
3*(2,1) - 1*(1,3) = (6,3) - (1,3) = (5,0)
```

对应格向量是 `(5,0)`。

## 2. 行列式

```text
det [[4,1],[1,3]] = 4*3 - 1*1 = 11
```

几何意义是基本平行四边形面积为 `11`，也可以理解为二维平面中这个格的点密度由该面积控制。

## 3. 短向量枚举

对 `[(4,1),(1,3)]` 枚举小系数，非零短向量之一是：

```text
(1,3)
```

它的平方范数是 `1^2 + 3^2 = 10`。另一个方向 `-(1,3)` 同样短。在这个系数范围内还会看到 `(3,-2)`，平方范数是 `13`。

## 4. 好基和坏基

同一个格可以通过 unimodular 变换得到不同基。比如标准格 `Z^2` 有基：

```text
(1,0), (0,1)
```

也有基：

```text
(1,0), (100,1)
```

第二组基仍生成 `Z^2`，因为可以用整数线性组合得到 `(0,1)`，但它非常倾斜，不适合最近点近似。

## 5. LLL 条件

size reduction 通常希望 `|mu[i][j]| <= 1/2`。直觉是：如果第 `i` 个基向量在第 `j` 个 Gram-Schmidt 方向上的投影太大，就减去一个整数倍的较早基向量，让它更短、更接近正交。

## 6. LWE 无噪声情形

如果 `e_i=0`，样本就是普通线性方程：

```text
b_i = <a_i, s> mod q
```

收集足够多线性无关样本后，可以用模线性代数解出 `s`。所以没有噪声的 LWE 不安全。

## 7. Toy LWE 正确性

因为 `b = A s + e`，所以：

```text
v - <s,u>
= b^T r + bit*floor(q/2) - <s, A^T r>
= (A s + e)^T r + bit*floor(q/2) - s^T A^T r
= e^T r + bit*floor(q/2)
```

如果 `e^T r` 足够小，就能判断结果更接近 `0` 还是 `floor(q/2)`，从而恢复 bit。

## 8. 参数失败

Toy 解码通常用 `q/4` 附近作为安全间隔。`q=97` 时，`q/4` 约为 `24.25`。如果累积噪声绝对值超过这个量，编码 `0` 可能看起来像 `1`，编码 `1` 也可能被拉回 `0`，因此解密失败。

## 9. 暴力攻击复杂度

候选数量是：

```text
q^n
```

当 `n=4, q=97` 时：

```text
97^4 = 88,529,281
```

这对教学脚本已经偏大，对真实参数则完全不可行。

## 10. SIS 与签名

SIS 要找满足公开线性关系的短向量。签名可以设计成：公钥给出某个矩阵或承诺关系，签名者凭 secret/trapdoor 找到一个短向量作为响应，验证者检查它确实短且满足公开方程。攻击者即使知道公开关系，也难以伪造短解。

## 11. Ring-LWE 效率

普通 LWE 需要显式存储大矩阵。Ring-LWE 用多项式环结构表示循环或负循环卷积，一个多项式就能隐式表达很多线性关系，因此公钥和计算都更紧凑。

## 12. Module-LWE 折中

普通 LWE 假设结构少，但密钥和计算较大。Ring-LWE 很高效，但环结构更强。Module-LWE 在多个环元素组成的模块上工作，效率接近 Ring-LWE，同时结构假设比单一 Ring-LWE 更温和。

## 13. KEM 和 PKE

PKE 接口通常是：

```text
encrypt(pk, message) -> ciphertext
decrypt(sk, ciphertext) -> message
```

KEM 接口通常是：

```text
encaps(pk) -> ciphertext, shared_secret
decaps(sk, ciphertext) -> shared_secret
```

TLS 需要建立对称会话密钥，而不是直接用公钥加密大量数据，所以 KEM 接口更自然。

## 14. ML-KEM 和 ML-DSA

ML-KEM 用于密钥封装，解决密钥建立问题。ML-DSA 用于数字签名，解决认证和完整性问题。它们都在 module-lattice 结构上构造，使用多项式环模块来获得效率和可分析的安全性。

## 15. FHE 噪声增长

格基 FHE 密文里带噪声。加法和乘法会让噪声增长，噪声超过阈值后就无法正确解密。Bootstrapping 的作用是同态地执行一次解密电路，把高噪声密文刷新成低噪声密文，从而支持更深的计算。
