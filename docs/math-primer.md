# 初学者数学基础

格基密码学看起来像密码学，其实入口处最重要的是三块数学：

1. 线性代数：向量、矩阵、基、线性组合、范数、行列式。
2. 模运算：所有数都在 `mod q` 的世界里循环。
3. 概率和噪声：LWE 的安全性来自“线性方程被小错误扰动”。

这份文档只讲够用的部分。读完后，你应该能顺利理解 `docs/concepts.md` 和 toy LWE 代码。

## 1. 标量、向量和矩阵

### 标量

标量就是一个数，例如：

```text
3, -7, 42, 97
```

在密码学里，标量经常会被放在模 `q` 的世界中，例如 `97` 下的整数集合：

```text
Z_97 = {0, 1, 2, ..., 96}
```

### 向量

向量是一串数。二维向量可以画成平面上的箭头，三维向量可以画成立体空间中的箭头：

```text
v = (2, 3)
w = (4, -1, 7)
```

向量可以表示：

- 空间中的位置。
- 一次移动。
- 多个变量组成的秘密值。
- 一行线性方程的系数。

在 LWE 中，secret 通常就是一个向量：

```text
s = (s1, s2, ..., sn)
```

### 矩阵

矩阵是按行和列排列的数：

```text
A = [ 4  1  7
      2  9  0 ]
```

它可以被理解为：

- 很多行向量放在一起。
- 一个线性变换。
- 很多线性方程的系数表。

在 LWE 中，公开矩阵 `A` 乘以秘密向量 `s`，再加噪声，得到公开向量 `b`：

```text
b = A*s + e mod q
```

## 2. 向量加法和数乘

向量加法按坐标相加：

```text
(2, 3) + (5, -1) = (7, 2)
```

数乘是每个坐标都乘上同一个数：

```text
3 * (2, 3) = (6, 9)
```

格的定义正是建立在这两个操作上：

```text
z1*b1 + z2*b2 + ... + zn*bn
```

这里 `z1, z2, ..., zn` 必须是整数。

## 3. 线性组合

线性组合就是“若干向量乘系数后相加”。

例如：

```text
b1 = (2, 1)
b2 = (1, 3)
3*b1 - 1*b2 = 3*(2,1) - (1,3) = (5,0)
```

如果系数允许是任意实数，线性组合可以填满一条线、一个平面或更高维空间。  
如果系数只能是整数，线性组合通常只得到离散点集，这就是整数格。

## 4. 点积

点积把两个同维向量变成一个数：

```text
<a, s> = a1*s1 + a2*s2 + ... + an*sn
```

例子：

```text
a = (7, 3, 11)
s = (4, 8, 1)
<a, s> = 7*4 + 3*8 + 11*1 = 63
```

LWE 的每个样本都长得像：

```text
b = <a, s> + e mod q
```

如果没有 `e`，这就是普通线性方程。`e` 是小噪声，但它会让恢复 `s` 变难。

## 5. 范数和距离

向量的长度叫范数。最常见的是欧几里得范数：

```text
||(x, y)|| = sqrt(x^2 + y^2)
```

很多时候只比较长度大小，可以用平方范数，避免开方：

```text
||(x, y)||^2 = x^2 + y^2
```

例子：

```text
||(3, 4)|| = 5
||(3, 4)||^2 = 25
```

SVP 的“最短向量”就是用范数衡量的。CVP 的“最近点”也是比较目标点和格点之间的距离。

## 6. 基

在线性代数里，一组基向量可以用来表示空间中的任意向量。  
在整数格里，格基是一组向量，允许用整数系数组合出格点。

重要区别：

- 线性代数中的基通常允许实数系数。
- 格中的基只允许整数系数。

所以：

```text
0.5*b1 + 0.3*b2
```

一般不是格点，因为系数不是整数。

## 7. 同一个格可以有很多基

这是格密码最重要的直觉之一。

例如标准二维整数格 `Z^2` 可以由下面这组好基生成：

```text
(1, 0), (0, 1)
```

也可以由下面这组坏基生成：

```text
(1, 0), (100, 1)
```

第二组基仍然生成同一个格，因为：

```text
(0, 1) = (100, 1) - 100*(1, 0)
```

好基短、接近正交；坏基长、倾斜。  
在某些早期格密码直觉中，私钥可以理解为好基，公钥可以理解为坏基。真实现代方案更复杂，但这个图像非常有帮助。

## 8. 行列式

二维矩阵：

```text
B = [ a b
      c d ]
```

它的行列式是：

```text
det(B) = a*d - b*c
```

几何意义：由两个基向量张成的平行四边形面积。

例如：

```text
B = [ 4 1
      1 3 ]
det(B) = 4*3 - 1*1 = 11
```

所以这个格的基本平行四边形面积是 `11`。面积越大，格点越稀疏；面积越小，格点越密。

## 9. Gram-Schmidt 正交化

Gram-Schmidt 的目标是把一组向量拆成“逐步去掉重叠方向”的正交成分。

在 LLL 中，我们不一定真的要得到整数格基，而是用 Gram-Schmidt 来衡量：

- 当前基向量之间是否太倾斜。
- 后面的向量在前面向量方向上的投影是否太大。
- 是否需要做 size reduction 或交换向量。

直觉：

```text
原始基：方向纠缠在一起
正交化成分：把每个新方向中“前面已经解释过的部分”去掉
```

LLL 反复使用这个信息，把坏基变成比较好的基。

## 10. 模运算

`mod q` 的意思是只看除以 `q` 的余数。

例如在 `mod 7` 中：

```text
8 = 1 mod 7
15 = 1 mod 7
-1 = 6 mod 7
```

加法和乘法之后都要取模：

```text
5 + 6 = 11 = 4 mod 7
5 * 6 = 30 = 2 mod 7
```

LWE 中所有公开样本通常都在 `Z_q` 中：

```text
b = <a, s> + e mod q
```

模运算带来两个效果：

- 数值被限制在固定范围内，便于计算机表示。
- 方程会“绕回去”，几何上像在一个循环空间中计算。

## 11. 中心化模表示

有时候 `96 mod 97` 更自然地看作 `-1`，因为它离 `0` 很近。

把 `0, 1, ..., q-1` 映射到大约 `(-q/2, q/2]` 的过程叫中心化：

```text
96 mod 97 -> -1
95 mod 97 -> -2
48 mod 97 -> 48
```

Toy LWE 解密时，我们会计算：

```text
phase = v - <s, u> mod q
```

然后把它中心化，看它接近 `0` 还是接近 `q/2`。

## 12. 噪声

LWE 中的噪声 `e` 通常很小，例如从 `{-1, 0, 1}` 或离散高斯分布中采样。

噪声的作用很微妙：

- 没噪声：线性代数直接解出 secret，不安全。
- 噪声太小：可能被攻击利用，不够安全。
- 噪声合适：恢复 secret 很难，解密仍然可靠。
- 噪声太大：解密失败率上升。

这就是格基密码参数设计的核心张力。

## 13. 概率分布

密码算法经常说“从某个分布采样”。这表示输出不是固定的，而是按规则随机产生。

常见例子：

- 均匀分布：每个值概率一样。
- 小噪声分布：`0` 概率高，`1`、`-1` 概率低一些。
- 离散高斯分布：整数上的钟形分布。

真实格基密码非常重视采样，因为分布不对可能泄露信息或导致安全证明失效。

## 14. 复杂度直觉

很多格问题在低维可以枚举解决。  
如果维度是 `n`，每个坐标有 `q` 种可能，暴力搜索大约是：

```text
q^n
```

例子：

```text
17^2 = 289
97^4 = 88,529,281
3329^256 约等于一个天文数字
```

这就是为什么 toy 参数能被脚本秒破，而真实参数不能靠枚举攻击。

## 15. 从基础到格密码的桥

把上面的概念串起来：

```text
向量和矩阵
  -> 线性方程
  -> 模 q 线性方程
  -> 加上小噪声
  -> LWE
  -> 公钥加密、KEM、签名、FHE
```

另一条线：

```text
整数线性组合
  -> 格
  -> 好基/坏基
  -> SVP/CVP/BDD
  -> LLL/BKZ
  -> 安全性估计与密码分析
```

如果这两条线都能看懂，就已经跨过了格基密码学最难的入门门槛。
