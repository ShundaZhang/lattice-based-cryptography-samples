# 核心概念清单

如果你是第一次接触格基密码，建议先读：

1. [初学者数学基础](math-primer.md)
2. [图形化理解格基密码](visual-guide.md)
3. 本文档

本文档的目标不是给最形式化的定义，而是帮你建立“看到一个术语时，知道它在系统里扮演什么角色”的地图。

## 0. 先建立三个总直觉

### 直觉一：格是离散空间

普通平面是连续的，任意两个点之间还有无限多点。格不是这样。格像规则重复的点阵，只能落在某些点上。

密码学利用这个离散性制造困难问题：

- 找一个短点。
- 找离目标最近的点。
- 从被噪声扰动的点恢复原始点。

在二维图上这些问题看起来很简单，但维度变成几百、上千时，直觉会失效。

### 直觉二：基是坐标系统

格基不是格本身，而是描述格的一套坐标系统。同一个格可以有很多不同基。

好基像一把清楚的尺子：短、方向分明、接近正交。
坏基像一把扭曲的尺子：长、倾斜、坐标解释困难。

LLL/BKZ 这类算法就是试图把坏基变好。格密码的安全参数必须抵抗攻击者能做到的规约程度。

### 直觉三：LWE 是“带小错误的线性代数”

没有噪声时：

```text
b = A*s mod q
```

这是线性代数，收集足够多方程就能解。
有噪声时：

```text
b = A*s + e mod q
```

每条方程都差一点点，于是恢复 `s` 变得困难。这个困难性支撑了很多现代后量子方案。

## 1. 整数格

给定线性无关向量 `b1, ..., bn`，它们生成的格是：

```text
L(B) = { z1*b1 + z2*b2 + ... + zn*bn | zi in Z }
```

矩阵 `B` 的行或列可以看作格基。本仓库代码统一把基向量当作矩阵的行。

同一个格有无穷多组基。好基一般短且接近正交，坏基通常很长、很倾斜。许多格算法的目标就是把坏基规约成较好的基。

初学者可以把 `b1, b2` 想成“走路规则”：

- 每次可以沿着 `b1` 走整数步。
- 每次可以沿着 `b2` 走整数步。
- 不允许走半步或 `0.3` 步。

所以 `(2*b1 - 3*b2)` 是格点，`(0.5*b1 + b2)` 通常不是格点。

密码学里常见的是高维格。二维格适合画图，高维格适合制造困难。

## 2. 基本域与行列式

基本域是基向量张成的平行多面体。满秩格的行列式等于基本域体积：

```text
det(L) = |det(B)|
```

行列式越小，单位体积内格点越密。许多安全估计会把维度、行列式和最短向量估计联系起来。

二维时，基本域是平行四边形；三维时是平行六面体；更高维时叫平行多胞体。它们的体积都由 determinant 给出。

为什么 determinant 有用？

- 它告诉我们格点整体有多密。
- 它不依赖于选择哪一组基。
- 它经常出现在 Minkowski 定理、Gaussian heuristic 和安全估计中。

## 3. 最短向量问题 SVP

SVP 要找格中非零最短向量：

```text
find v in L, v != 0, minimizing ||v||
```

精确 SVP 在高维很难。密码学通常依赖近似版本或相关问题的困难性。

二维图上，SVP 看起来就是“找离原点最近的非零黑点”。
高维里问题变难，因为候选方向太多，局部看起来短的方向未必是真正最短。

常见变体：

- `SVP_gamma`：找长度不超过最短向量 `gamma` 倍的向量。
- SIVP：找一组足够短且线性无关的向量。
- uSVP：最短向量明显比第二短向量短。

## 4. 最近向量问题 CVP

CVP 给定目标点 `t`，找最近格点：

```text
find v in L minimizing ||t - v||
```

CVP 通常比 SVP 更难。Babai nearest plane 是一个经典近似算法，它依赖较好的格基。

CVP 可以想象成导航问题：

```text
给定目标 t，地图上只有格点可以停靠。要找离 t 最近的停靠点。
```

如果手里有好基，近似最近点比较容易。
如果只有坏基，目标点附近的格点关系会很难判断。

## 5. 有界距离解码 BDD

BDD 是 CVP 的特殊情形：目标点保证离某个格点足够近。LWE 解密可以看作一种带结构的 BDD：密文含有接近某个格点的噪声扰动，私钥帮助把它解码回来。

BDD 比 CVP 多了一个承诺：目标点不是随便来的，它离某个正确格点很近。

这和纠错码很像：

- 发送的是合法码字。
- 传输中加了一点错误。
- 接收者把收到的点纠正回最近码字。

LWE 解密也是类似逻辑：噪声小于阈值时能恢复，噪声超过阈值时会失败。

## 6. LLL 规约

LLL 是 Lenstra、Lenstra、Lovasz 提出的多项式时间格基规约算法。它通过两类操作改善基：

- size reduction：让后面的基向量减少在前面基向量方向上的投影。
- Lovasz condition：如果局部顺序不够好，就交换相邻基向量。

LLL 不能保证找到真正最短向量，但在密码分析和数论计算中非常有用。

LLL 的输入是一组整数基，输出仍然是同一个格的一组整数基。它不会改变格点集合，只改变描述方式。

你可以把 LLL 想成“整理坐标轴”：

```text
坏基 -> 反复减去整数倍、交换相邻向量 -> 较短且较正交的基
```

LLL 是理解现代 BKZ 的台阶。BKZ 更强，但也更贵。

## 7. BKZ 规约

BKZ 可以看作更强的块版本 LLL。它每次在一个维度为 `beta` 的块内调用 SVP oracle 或枚举算法。`beta` 越大，规约越强，成本也越高。现代格密码安全估计常讨论攻击者能承受多大的 BKZ block size。

安全估计里常会问：

```text
攻击者需要多大的 block size 才能破这个参数？
这个 block size 对应多少计算成本？
```

这就是为什么真实方案不会只说“维度很高所以安全”，而是需要结合具体攻击模型和参数估计。

## 8. SIS

SIS，Short Integer Solution，给定随机矩阵 `A in Z_q^{n x m}`，找一个短向量 `x` 使得：

```text
A x = 0 mod q
```

SIS 常用于哈希、承诺、签名方案。直觉上，模方程容易有很多解，但要求解很短会变难。

SIS 的关键是“短”。如果不要求短，模线性方程往往有很多解。
要求短之后，问题变成在巨大解空间中找一个小向量。

密码构造中的直觉：

- 哈希：输出相同意味着找到短关系。
- 承诺：打开承诺需要给出满足关系的短向量。
- 签名：签名像一个短响应，验证者检查公开方程和短范数。

## 9. LWE

LWE，Learning With Errors，给定样本：

```text
(a_i, b_i = <a_i, s> + e_i mod q)
```

其中 `s` 是 secret，`e_i` 是小噪声。任务是恢复 `s`，或判断这些样本和均匀随机样本是否可区分。

噪声是 LWE 的灵魂：

- 没有噪声时，就是普通线性方程组。
- 噪声太小时，容易被攻击。
- 噪声太大时，解密容易失败。

LWE 有两种常见表述：

- Search-LWE：恢复 secret `s`。
- Decision-LWE：判断样本来自 LWE，还是完全随机。

许多密码系统依赖 decision 版本：如果密文看起来和随机数差不多，攻击者就很难区分消息。

LWE 的参数包括：

- `n`：secret 维度。
- `m`：样本数量。
- `q`：模数。
- 噪声分布：错误项的大小和概率。

这些参数互相牵制。增大维度通常提高安全性，增大噪声也可能提高安全性，但噪声太大又会造成解密失败。

## 10. Ring-LWE 与 Module-LWE

LWE 的矩阵很大。Ring-LWE 把向量和矩阵结构放进多项式环，例如：

```text
Z_q[x] / (x^n + 1)
```

Module-LWE 位于普通 LWE 和 Ring-LWE 之间，保留较高效率，同时避免过强的环结构假设。ML-KEM 和 ML-DSA 都使用 module-lattice 风格结构。

为什么要从 LWE 走向 Ring/Module？

普通 LWE 的矩阵很大，存储和计算成本高。
Ring-LWE 用多项式乘法表达大量线性关系，能大幅压缩矩阵。
Module-LWE 则在效率和结构假设之间折中，是当前标准方案的重要基础。

可以粗略记：

```text
LWE: 结构少，东西大
Ring-LWE: 结构多，效率高
Module-LWE: 中间路线，现代标准常用
```

## 11. NTRU

NTRU 使用多项式环中的短多项式和卷积乘法。它历史很长、速度快，也启发了 Falcon/FN-DSA 这类基于 NTRU lattice 的签名。

NTRU 的高层图像是：

- 私钥是短多项式。
- 公钥是某种模多项式商。
- 安全性可以关联到 NTRU lattice 中找短向量的困难性。

Falcon/FN-DSA 使用 NTRU lattice 和高斯采样思想，签名很紧凑，但实现复杂度高。

## 12. KEM

KEM，Key Encapsulation Mechanism，不直接加密任意长消息，而是封装一个共享密钥：

```text
keygen -> pk, sk
encaps(pk) -> ciphertext, shared_secret
decaps(sk, ciphertext) -> shared_secret
```

现代混合加密通常用 KEM 建立对称密钥，再用 AEAD 加密数据。

KEM 不直接加密“你好世界”这样的消息。它只负责让双方得到同一个随机共享密钥。

为什么这很实用？

- 对称加密非常快，适合大量数据。
- 公钥算法只做密钥建立，接口清晰。
- TLS 这类协议本来就需要先建立会话密钥。

## 13. 数字签名

格基签名常见路线：

- Fiat-Shamir with aborts：Dilithium/ML-DSA 的核心风格。
- hash-and-sign with trapdoor：GPV/Falcon 思想相关。
- SIS 派生签名：签名是满足公开关系的短向量。

签名要解决两个问题：

- 只有私钥持有者能产生有效签名。
- 任何人拿公钥都能验证。

格基签名常把“能找到短向量/短响应”作为私钥能力，把“检查短且满足公开关系”作为验证逻辑。

## 14. 全同态加密

全同态加密允许在密文上计算。格基 FHE 的核心主题包括：

- 噪声增长。
- 模数切换。
- relinearization。
- bootstrapping。
- BFV/BGV 适合精确整数或模运算。
- CKKS 适合近似实数和机器学习推理。

FHE 可以理解为“密文上的计算器”。
你把加密数据交给云端，云端不知道明文，却能算出加密结果。

核心难点是噪声增长：

- 加法让噪声增加一点。
- 乘法让噪声增加更多。
- 噪声太大就不能正确解密。
- bootstrapping 用一次同态解密过程刷新噪声。

## 15. 后量子标准中的位置

当前最重要的格基标准方向：

- ML-KEM：密钥封装，基于 module-lattice 思想，源自 Kyber。
- ML-DSA：数字签名，基于 module-lattice 思想，源自 Dilithium。
- FN-DSA：签名方向，基于 NTRU lattice/Falcon 思想。

需要注意：

- SLH-DSA/SPHINCS+ 是后量子签名标准，但不是格基方案。
- HQC 是 code-based KEM，也不是格基方案。
- 后量子密码不是只有格基，但格基是当前最核心、最实用的路线之一。
